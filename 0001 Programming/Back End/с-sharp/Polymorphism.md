# <center>Constructors</center> 

> [!blank-container|right] 
> ## Definition  
> [[Polymorphism|Polymorphism]] ‚Äî a principle of [[OOP|object-oriented programming]]
> that allows objects of **different classes** to respond differently to the same method calls.  
>
> Types of polymorphism in C#:  
> - [[Virtual Methods|Virtual methods]] and `override` ‚Äî redefine behavior in child classes  
> - [[Abstract Classes|Abstract classes]] ‚Äî force child classes to implement methods  
> - [[Interfaces|Interfaces]] ‚Äî different classes can implement the same behavior by contract  
>
> ## Characteristics  
> - Provides flexibility and extensibility of code  
> - Achieved via `virtual`, `override`, `abstract`, and `interface`  
> - Enables working with objects through a common type (base class or interface)  
> ## [[Upcasting –∏ Downcasting]]  
> - **Upcasting** ‚Äî safe conversion of a child class object to its parent type  
> - **Downcasting** ‚Äî explicit conversion of a parent type object back to a child type  
>
